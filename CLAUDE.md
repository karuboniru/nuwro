# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What is NuWro

NuWro is a Monte Carlo neutrino event generator simulating neutrino-nucleon and neutrino-nucleus reactions from threshold to TeV energies. It uses ROOT for event storage and requires PYTHIA6 for DIS.

## Build

**Prerequisites**: ROOT with PYTHIA6, `$ROOTSYS` and `$ROOTEGPythia6_ROOT` environment variables set.

```bash
make          # build all targets into bin/
make clean    # remove object files
make distclean  # remove object files and executables
```

CMake alternative:
```bash
mkdir build && cd build && cmake .. && make
```

The Makefile generates two intermediate files that must not be manually edited:
- `src/params_all.h` — auto-generated from `src/params.xml` via `src/params.sed`
- `src/event1Dict.cc` / `src/event1Dict.h` — ROOT dictionary generated by `rootcint`

## Running

```bash
cp data/params.txt .       # copy template config to working directory
# edit params.txt as needed
nuwro                      # run event generation (reads params.txt)
myroot eventsout.root      # analyze output interactively
```

## Architecture

### Event generation flow

`main.cc` → `NuWro::main()` (nuwro.cc):
1. Parse `params.txt` config
2. Initialize detector geometry (`geomy`), beam, and nucleus target
3. Select event generation mode:
   - `test_events()` + `real_events()` — standard two-phase unweighted generation: a test run estimates per-channel max weights and cross sections via `Chooser`; the real run uses rejection sampling against those max weights, producing unweighted events (all stored with `e->weight = total_xsec`)
   - `real_events_mh()` — Metropolis-Hastings unweighted generation; skips the test run, learns channel cross sections adaptively during sampling
   - `user_events()` — analysis mode

For each event: choose interaction channel via `Chooser` (cross-section weighted), generate kinematics, apply FSI via `kaskada7.cc`.

### Physics processes

Enabled/disabled per-run via flags in `params.txt` (`dyn_qel_cc`, `dyn_res_cc`, `dyn_dis_cc`, `dyn_coh_cc`, `dyn_mec_cc`, `dyn_hyperon`, etc.):

| Process | Key files |
|---------|-----------|
| QEL (quasi-elastic) | `src/qelevent1.cc`, `src/qel_sigma.cc` |
| RES (resonance) | `src/dis/resevent2.cc`, `src/dis/res_xsec.cc` |
| DIS (deep inelastic) | `src/dis/disevent.cc`, `src/dis/dis_cr_sec.cc` |
| COH (coherent) | `src/cohevent2.cc`, `src/cohdynamics2.cc` |
| MEC (meson exchange) | `src/mecevent*.cc`, `src/mecdynamics*.cc` |
| HYP (hyperon) | `src/hypevent.cc`, `src/hyperon_*.cc` |
| FSI (final state int.) | `src/kaskada7.cc`, `src/scatter.cc` |

Multiple MEC models are compiled in: `mecevent.cc` (model 1), `mecevent2.cc` (model 2), `mecevent_tem.cc` (TEM), `mecevent_Nieves.cc`, `mecevent_SuSA.cc`, `mecevent_2020Valencia.cc` — selected by `mec_model` param.

### Event data structure

`event1.h` defines the `event` class (ROOT `TObject`):
- `in` — incoming particles
- `out` — outgoing particles before FSI
- `post` — outgoing particles after FSI
- `flags` struct — categorizes interaction type (`qel`, `res`, `dis`, `coh`, `mec`, `hyp`, `lep`)
- Cross-section weights and nuclear information stored per event

### Parameters system

`src/params.xml` defines all parameters with defaults. `src/params.sed` transforms the XML into C++ macro calls, producing `src/params_all.h`. The `params` struct (via `src/params.h`) exposes all parameters to C++ code. The user-facing config is `data/params.txt`.

### Spectral functions

`src/sf/` contains data-driven nucleon momentum distributions for specific nuclei (C12, Ca40, Ca48, Fe56) as an alternative to the default Fermi gas model. Selected via `nucleus_target` and `nucleus_sf` params.

### Reweighting framework

`src/rew/` provides post-generation event reweighting without re-running the generator. Key files: `Reweighters.cc`, `rewQEL.cc`, `rewRES.cc`, `rewNorm.cc`. Executables: `reweight_to` and `reweight_along`.

### Metropolis-Hastings sampler

`src/metropolis.h` provides a template-based M-H sampler. `src/generatormt.cc` is the underlying MT19937 RNG. `NuWro::real_events_mh()` in `nuwro.cc` uses this as an alternative to the standard rejection-sampling path: it learns channel cross sections adaptively and does not require a separate test run. See `metropolis-hastings.md` for details.
